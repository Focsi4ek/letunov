# Конспект 2. История развития сетей и сетевые модели

## 1. История развития компьютерных сетей

**Ключевые даты и события:**

* **Начало 70‑х** — появление первых нестандартных локальных сетей.
* **1971** — Рей Томлинсон (BNN) написал первую программу для работы с электронной почтой.
* **1973** — Роберт Меткалф (Xerox) предложил идею и название технологии **Ethernet**.
* **1974** — Винтон Серф и Роберт Кан разработали протокол управления передачей **TCP**.
* **1978–1979** — разработка эталонной модели **OSI**.
* **Начало 1980‑х**:
    * создание стека протоколов **TCP/IP** (независимость компьютера от сети);
    * развёртывание TCP/IP на всех узлах ARPANET;
    * формирование сети Интернет в современном виде.
* **2 ноября 1988** — червь Морриса поразил около 6 000 узлов ARPANET, практически выведя сеть из строя. Результат: создание **CERT** (Computer Emergency Response Team).
* **Середина 80‑х** — разработка стандартных технологий локальных сетей:
    * 1980 — **Ethernet**;
    * 1985 — **Token Ring**;
    * 1985 — **FDDI**.
* **Конец 1980‑х — начало 1990‑х** — замена ARPANET на **NSFNet** (сеть Национального научного фонда NSF).
* **1991–1992** — Тим Бернерс‑Ли и коллеги (CERN) изобрели технологию **World Wide Web**.
* **1994** — создание протокола **PPP**.
* **1995** — замена NSFNet на коммерческую опорную сеть, появление **ISP** (Internet Service Providers). Переход к современному статусу сети Интернет.



## 2. Сетевые модели и протоколы

### Основные понятия

* **Коммуникационный протокол** — формализованный набор правил взаимодействия узлов сети.
* **Стек протоколов** — иерархически организованный набор протоколов для взаимодействия узлов в сети.

### Эталонная модель OSI (Open System Interconnection)

Разработана в начале 80‑х ISO как международный стандарт архитектуры компьютерной сети. Определяет уровни взаимодействия в сетях с коммутацией пакетов.

**Уровни OSI (снизу вверх):**

1. **Физический** (Physical Layer):
    * функция: передача потока битов по физическим каналам связи (например, витая пара);
    * реализуется на всех устройствах, подключённых к сети;
    * пример протокола: спецификация 100Base‑T4 стандарта Ethernet.
2. **Канальный** (Data Link Layer):
    * первый уровень с коммутацией пакетов;
    * PDU (Protocol Data Unit) — кадр (frame);
    * функции: доставка кадра между узлами сети (LAN) или соседними узлами (WAN, PPP, HDLC);
    * задачи: обнаружение и коррекция ошибок, проверка доступности среды передачи данных (MAC);
    * реализуется адаптерами, мостами, коммутаторами, маршрутизаторами.
3. **Сетевой** (Network Layer):
    * служит для межсетевого взаимодействия (internetworking);
    * PDU — пакет;
    * функции: физическое объединение сетей, определение маршрута;
    * примеры протоколов: IP, IPX, RIP, BGP.
4. **Транспортный** (Transport Layer):
    * обеспечивает передачу данных с нужной степенью надёжности;
    * OSI определяет 5 классов транспортного сервиса (0–4);
    * PDU — сегмент или дейтаграмма;
    * задачи: реализация транспортного соединения, мультиплексирование/демультиплексирование, управление потоком;
    * примеры: TCP, UDP.
5. **Сеансовый** (Session Layer):
    * управляет взаимодействием сторон, синхронизирует сеанс;
    * пример: SSH.
6. **Представления данных** (Presentation Layer):
    * представляет информацию без изменения содержания;
    * задачи: отображение данных, шифрование/дешифрование (SSL), сжатие данных.
7. **Прикладной** (Application Layer):
    * доступ к разделяемым ресурсам и организация совместной работы;
    * PDU — сообщение (message);
    * примеры: FTP, SMB, NCP.

### Модель DoD (TCP/IP)

Четырёхуровневая модель сетевого взаимодействия, разработанная Министерством обороны США. Практическая реализация — стек протоколов TCP/IP.

**Уровни:**

1. **Уровень приложений** (объединяет прикладной, представления данных и сеансовый уровни OSI).
2. **Транспортный уровень** (соответствует транспортному уровню OSI).
3. **Межсетевой (Internet)** (соответствует сетевому уровню OSI).
4. **Уровень доступа к сети** (объединяет канальный и физический уровни OSI).

### Стандартные стеки коммуникационных протоколов

* OSI;
* TCP/IP;
* IPX/SPX;
* NetBIOS/SMB;
* DECnet;
* SNA.

## 3. Прикладной уровень

### Архитектура прикладных протоколов Internet

**Принципы построения:**

* **«Клиент‑сервер»** (веб‑сервер и браузер, TELNET‑сервер и клиент).
* **Peer‑to‑peer** (равноправные участники).

**Конечная точка коммуникации** задаётся с помощью **сокета** (интерфейс между прикладным и транспортным уровнями):

* имя хоста (символьное имя или IP‑адрес);
* номер порта (HTTP — 80, SMTP — 25 и т. д., заданы в RFC‑1700, RFC‑3232, www.iana.org).

**По принципу передачи данных:**

* текстовые протоколы;
* бинарные протоколы.

**Элементы прикладного протокола:**

* типы сообщений (запросы и ответы);
* синтаксис сообщений;
* семантика полей;
* правила генерации сообщений.

## 4. Система доменных имён (DNS)

**Домены:**

* плоские имена (для небольших сетей: NW1_1, MAIL2);
* доменная система имён (en.wikipedia.org, msdn.microsoft.com).

**Типы доменных имён:**

* краткое доменное имя;
* относительное доменное имя;
* полное доменное имя (FQDN — Fully Qualified Domain Name).

**Домены верхнего уровня:**

* родовые (com, edu, org и т. д.);
* домены государств (by, ru, uk, us и т. д.);
* sponsored‑домены (зарегистрированные на компанию/организацию).

**DNS (Domain Name System):**

* централизованная служба с распределённой базой записей ресурсов;
* запись ресурса: `DomainName Class Type Value` (например, тип A — IP‑адрес хоста, NS — сервер имён верхнего уровня, CNAME — псевдоним, MX — почтовый сервер);
* пространство имён поделено на зоны, обслуживаемые отдельными серверами;
* разрешение имён выполняет **резольвер DNS** (resolver).

**Схемы разрешения DNS‑имен:**

* рекурсивная (косвенная);
* итеративная (нерекурсивная);
* гибридная.

**Обратная зона** (in‑addr.arpa / ip6.arpa) — поиск DNS‑имени по IP‑адресу.

**Порядок разрешения имени в Windows:**

1. Проверка имени локального узла.
2. Кэш DNS (запросы + файл hosts).
3. Запрос DNS‑серверу.
4. Если имя не плоское — ошибка.
5. Иначе — NetBIOS имя в кэше.
6. Запрос WINS‑серверу.
7. Широковещательный запрос в подсеть (x3).
8. Файл lmhosts.
9. Ошибка.

**Расположение файла hosts:**

* Windows: `/Windows/system32/drivers/etc/hosts`;
* GNU/Linux: `/etc/hosts`.

## 5. Основные прикладные протоколы

### FTP (File Transfer Protocol)

* передача файлов между удалёнными компьютерами;
* идентификация: учётная запись или анонимный доступ;
* использует два TCP‑соединения: управляющее (порт 21) и передачи данных;
* описание: RFC 959.

### TFTP (Trivial FTP)

* упрощённая версия FTP;
* без системы безопасности и идентификации;
* базируется на UDP (порт 69);
* данные передаются блоками по 512 байт;
* используется для загрузки ОС в бездисковые станции и конфигурационных файлов в маршрутизаторы.

### Электронная почта (E‑mail)

**Модель X.400:**

* UA/MUA (User/Mail User Agent) — почтовый клиент;
* MTA (Message Transfer Agent) — агент передачи сообщений;
* MDA (Message Delivery Agent) — агент доставки сообщений;
* MS (Message Store) — банк сообщений.
