# Лабораторная работа №4
## Диагностика работоспособности персональных компьютеров

**Цель работы:** освоить методы диагностики работоспособности персонального компьютера с использованием специализированных программ-диагностик, провести комплексное тестирование аппаратных компонентов и оценить их текущее состояние и производительность.

## Часть 1. Результаты диагностики компьютера

Для проведения диагностики аппаратной части компьютера была использована программа PC Wizard — многофункциональный инструмент для анализа и тестирования компонентов ПК. Программа позволяет получить подробные сведения об оборудовании и провести нагрузочные тесты для оценки реальной производительности.

Диагностика выполнена в следующей последовательности:
1. Запуск PC Wizard с правами администратора.
2. Выбор необходимых тестов для отдельных компонентов.
3. Проведение тестирования в штатном режиме без разгона компонентов.
4. Фиксация результатов в автоматическом режиме.

## Часть 2. Ответы на контрольные вопросы

**1. Причины возникновения проблем с совместимостью программного обеспечения**

Проблемы совместимости ПО могут возникать по целому ряду причин, которые можно разделить на несколько категорий:

**Аппаратные причины:**
* несоответствие системных требований программы реальным характеристикам компьютера (недостаточный объём ОЗУ, слабая видеокарта, низкая частота процессора);
* отсутствие поддержки необходимых инструкций процессора (например, SSE4.2, AVX);
* несовместимость с типом накопителя (например, требования к NVMe SSD вместо обычного HDD);
* проблемы с периферийными устройствами (неподдерживаемые модели принтеров, сканеров и т. д.).

**Программные причины:**
* различия в версиях операционной системы (программа, написанная для Windows 7, может не работать на Windows 11);
* несовместимость разрядности (32‑битное приложение на 64‑битной системе или наоборот);
* устаревшие или повреждённые драйверы устройств;
* конфликты между разными версиями библиотек (DLL‑hell);
* изменения в API и системных вызовах между версиями ОС.

**Системные причины:**
* ограничения прав доступа (UAC в Windows, SELinux в Linux);
* защита системных файлов (Windows Resource Protection);
* политики безопасности организации;
* антивирусное ПО, блокирующее выполнение определённых операций.

**Пользовательские причины:**
* ручная модификация системных файлов или реестра;
* установка несовместимых патчей или модов;
* использование пиратского ПО с изменённым кодом;
* неправильная установка программы (пропущенные зависимости, повреждённые файлы).

**Сетевые причины:**
* требования к определённым сетевым протоколам или портам;
* необходимость постоянного интернет‑подключения;
* зависимость от облачных сервисов, которые могут быть недоступны.

**2. Методы уменьшения проблем с совместимостью программного обеспечения**

Для минимизации и устранения проблем с совместимостью ПО существует множество методов, которые можно применять как превентивно, так и для решения уже возникших проблем:

**Профилактические методы:**
* проверка системных требований перед установкой ПО;
* обновление ОС и драйверов до последних версий;
* создание точки восстановления системы перед установкой нового ПО;
* чтение отзывов и форумов о совместимости конкретной программы с вашей конфигурацией.

**Методы решения существующих проблем:**

1. **Использование режимов совместимости:**
   * запуск программы в режиме совместимости с более ранней версией ОС;
   * настройка параметров совместимости (отключение визуальных эффектов, запуск в режиме 640×480 и т. д.).

2. **Обновление компонентов среды выполнения:**
   * установка актуальных версий .NET Framework;
   * обновление Visual C++ Redistributable;
   * установка последней версии DirectX;
   * обновление Java Runtime Environment (если требуется).

3. **Работа с драйверами:**
   * обновление драйверов устройств до последних версий;
   * откат к предыдущей версии драйвера, если проблема возникла после обновления;
   * использование универсальных драйверов вместо специализированных.

4. **Изоляция приложений:**
   * запуск в виртуальной машине (VirtualBox, VMware, Hyper‑V);
   * использование контейнеров (Docker);
   * работа в песочнице (Sandboxie).

5. **Эмуляция среды:**
   * использование Wine для запуска Windows‑приложений в Linux;
   * применение эмуляторов старых систем (DOSBox, QEMU).

6. **Настройка системы:**
   * временное отключение UAC для установки проблемного ПО;
   * изменение политик безопасности;
   * предоставление программе прав администратора.

7. **Альтернативные решения:**
   * поиск аналогов программы с лучшей совместимостью;
   * использование веб‑версий приложений вместо десктопных;
   * переход на облачные сервисы.

8. **Восстановление системы:**
   * откат к точке восстановления;
   * восстановление целостности системных файлов (`sfc /scannow`);
   * переустановка проблемных компонентов.

9. **Диагностика и анализ:**
   * использование инструментов диагностики совместимости (Compatibility Administrator);
   * анализ логов событий Windows;
   * мониторинг ресурсов системы во время работы программы.

10. **Профилактическое обслуживание:**
    * регулярная очистка системы от временных файлов;
    * дефрагментация HDD (если используется);
    * проверка состояния SSD (S.M.A.R.T.);
    * обновление BIOS/UEFI до последней версии.
